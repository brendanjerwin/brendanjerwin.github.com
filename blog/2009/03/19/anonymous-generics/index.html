C# Generics Using Anonymous Types

Mar 19th, 2009

EDIT: After some discussion it was decided that the static factory class example is much clearer. So I’ve rearranged things to promote it over the static factory method in the generic type method.

I was trying to find a better way to send multiple models to my views in an ASP MVC project and discovered a pretty neat trick. It didn’t solve my MVC problem but I still think it’s pretty cool.

Using a technique that seems to be called casting by example and a static factory method, it is possible to instantiate a generic class using an anonymous type. The resulting instance, due to the magic of type inference, has full type safety and intellisense.

The syntax for creating an instance is reminiscent of using the new “initializer” syntax: var foo = MyClass.Build(new {Name="This is a Name", Count=1});
