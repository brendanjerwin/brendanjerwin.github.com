Html.ListBoxFor - Yes, It Needs Two Lists

Feb 10th, 2010

It took me a while to figure this out. As a matter of fact, this is the second post on the subject since the first post was wrong.

In ASP.NET MVC2, the Html.ListBoxFor() method needs two parameters that seem to refer to the same data:


(My example is concerned with a Permission enumeration.)

The behavior of this method just wasn’t very clear at all. It turns out that the first parameter, expression, is intended to reference the model property that contains a list of the selected items. In my case its an IEnumerable<Permission>. Call this parameter “selected items” instead of expression.

The second parameter is an IEnumerable<SelectListItem> called selectList. It is a list of SelectListItems which represents all the possible values that could be selected. Call this parameter “all available items” instead of selectList.

When ListBoxFor() is rendering the list html, it will mark the items as selected when the item’s value is found in both lists. It’s important to not use an IEnumerable<SelectListItem> on the model property.

When the model binders deserialize the form on a POST, the model’s property will only contain entries for those values that were marked as selected.

Once I figured out what the intent was I’m pretty happy with this helper, but boy those parameter names are useless!
