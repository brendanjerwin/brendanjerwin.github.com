Nubular (Nu) Command - Specification

Sep 6th, 2010

UPDATE: Released in v0.2.5.

Nubular (Nu) will be getting a new command soon: specification.

The specification command is intended to provide detailed package information to integrating applications. (Most notably NuForVS.)

It can be used to get package information from a lib folder (default), the local machine’s gem cache (--cache), or from the configured gem servers (--remote).

When querying either the local cache or remote servers, you can specify a particular version with the --version switch, or Nu will simply return the specification for the highest, non-pre-release, version available. It’s not valid to specify a version when querying against a lib folder.

The specification command always returns JSON even if the --json switch is not set, so don’t be surprised if you can’t read the output.
